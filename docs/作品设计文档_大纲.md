## 封面

- 作品名称：智慧伴学 EduMate — 基于大模型的个性化资源生成与学习多智能体系统
- 赛题：第十五届中国软件杯大赛 A组
- 赛道/赛项：A-ST（按实际填写）
- 团队信息：学校、指导教师、团队成员
- 一句话定位：面向高等教育 Python 程序设计课程的“多智能体协同 + 认知风格自适应”个性化学习系统

---

## 摘要

- 背景：高等教育个性化学习需求与传统教学模式的矛盾
- 方案：智慧伴学以 Neo4j 知识图谱为硬约束， Orchestrator / Profiler / Navigator / Generator / Reviewer 五角色多智能体协同，结合 DeepSeek / 讯飞星火大模型，实现对话式画像、神经符号资源生成、辩论议会审核、BKT 学习评估闭环
- 效果：资源生成从 ~3s 降至 ~0.02s（缓存后），index chunk 从 465KB 降至 ~98KB，支持 33 个知识节点的 Python 程序设计课程
- 关键词：大模型、多智能体、个性化学习资源、知识图谱、BKT、认知风格自适应

## 一、目标问题与意义价值

### 1.1 应用领域
- 高等教育程序设计课程（以 Python 为例）
- 可扩展至人工智能、计算机、电子信息等相关专业课程

### 1.2 解决与关注的问题
- 学习资源繁杂无序，难以匹配学生个体差异
- 课堂集体讲授无法兼顾不同学习节奏
- 现有智能辅助系统缺乏多模态生成与多智能体协同
- 大模型生成内容易出现“幻觉”和超纲

### 1.3 实现目标与基本功能
- 对话式学习画像构建（≥6 维度）
- 多智能体协同的个性化资源生成（≥5 种类型）
- 个性化学习路径规划与资源推送
- 智能辅导（苏格拉底式对话 + 数字人助教）
- 学习效果评估与动态调整

### 1.4 理论意义与应用价值
- 理论意义：多智能体协同 + 神经符号约束在教育场景的落地
- 应用价值：提升学生学习效率、降低教师备课负担、实现因材施教数字化

### 1.5 现有方案局限性对比
- 传统 LMS：资源固定，无个性化
- 单一 LLM 对话工具：无知识体系约束，易幻觉
- 现有知识图谱系统：缺少动态画像与资源生成闭环

## 二、设计思路与方案

### 2.1 总体设计思路
- 以“学生画像”为中心，以“知识图谱”为约束，以“多智能体协作”为引擎
- 数据闭环：对话 → 画像 → 路径 → 资源 → 练习 → 评估 → 画像更新

### 2.2 系统总体架构
- 架构图：User → Web Frontend → FastAPI Backend → Agent Orchestrator → Profiler / Navigator / Generator / Reviewer
- 模块功能划分表

### 2.3 多智能体协同架构
- Orchestrator：意图识别、会话状态维护、SOP 路由
- Profiler：解析学生输入，更新认知画像
- Navigator：基于知识图谱规划学习路径
- Generator：生成个性化教学资源包
- Reviewer：4-Prompt 辩论议会审核 + BKT 评估

### 2.4 对话式学习画像构建
- 自然语言对话抽取 ≥6 维度：知识水平、认知风格（视觉/听觉/动觉）、场依存/独立、学习节奏、目标导向、易错点偏好
- 画像证据动态更新机制

### 2.5 神经符号约束与知识图谱
- Neo4j 知识图谱：前置依赖、难度等级、易错点、禁用知识点
- AST 校验防止超纲与幻觉
- 无 Docker 时自动降级为内存图

### 2.6 个性化资源生成与审核
- 资源类型：讲义、示例、练习、代码题、思维导图、拓展阅读
- Generator → Reviewer 辩论议会 → 输出审核通过资源
- 资源缓存机制（concept, profile_hash）

### 2.7 个性化学习路径规划
- A* / 拓扑排序路径规划
- 前置知识补齐 → 当前目标 → 后续拓展
- 动态根据掌握度调整路径

### 2.8 智能辅导
- 苏格拉底式对话（SocraticPanel）
- 数字人助教（DigitalHuman + Spark TTS）
- 多模态解答：文字、图解、代码示例

### 2.9 学习效果评估与 BKT
- BKT（Bayesian Knowledge Tracing）掌握度追踪
- 掌握度热力图
- 根据评估结果动态调整学习资源推送策略

### 2.10 内容安全与防幻觉机制
- 输入安全过滤
- 知识图谱硬约束
- AST 代码案例可运行性校验
- Reviewer 辩论议会审核

## 三、实现方案

### 3.1 技术栈选型
- 前端：React 18 + Vite + TypeScript + Tailwind CSS + shadcn/ui + Monaco + Pyodide
- 后端：FastAPI + Python 3.11+
- 大模型：DeepSeek / 讯飞星火 / Mock
- 多智能体：自定义 Agent Orchestrator
- 知识图谱：Neo4j Community / 内存图
