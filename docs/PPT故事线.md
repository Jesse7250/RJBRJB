---

## 第 1 部分：问题背景（2-3 页）

1. **封面**
   - 标题：智慧伴学 EduMate
   - 副标题：基于大模型的 Python 个性化学习多智能体系统
   - 团队、赛事、日期

2. **痛点页**
   - 传统平台：千人一面、缺乏实时反馈、资源质量不可控
   - 学生真实场景：学不懂、练不会、错因不明

3. **解决思路**
   - 用多智能体模拟“私人助教团队”
   - 用知识图谱做约束，用 BKT 做掌握度追踪
   - 用群体错误驱动资源进化

---

## 第 2 部分：系统架构（3-4 页）

4. **总体架构图**
   - 前端 React + 后端 FastAPI + DeepSeek API + SQLite
   - 5 角色执行层：Orchestrator / Profiler / Navigator / Generator / Reviewer

5. **5 角色分层说明**
   - Orchestrator：总调度
   - Profiler：画像构建
   - Navigator：路径规划
   - Generator：资源生成
   - Reviewer：审核 + 辅导 + 评估（内部 4-Prompt 辩论议会）

6. **数据流图**
   - 登录 → 画像 → 路径 → 资源生成 → 辩论审核 → 学习练习 → 评估反馈 → 知识熔炉重审
