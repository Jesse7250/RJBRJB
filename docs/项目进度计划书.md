## 一、项目目标

在 4 周内完成一个可运行、可演示、具备国一竞争力的 **Python 个性化学习多智能体系统**。系统以大模型为主推理引擎，以知识图谱为神经符号约束基础，实现对话式画像构建、多 Agent 协同资源生成、辩论议会质量审核、认知风格自适应渲染等核心功能。

## 六、风险与应对（已更新）

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| 大模型 API 不稳定 | 高 | 中 | 申请足额额度；本地缓存热门资源；准备离线降级模式 |
| Agent 响应时间过长 | 高 | 中 | 辩论议会异步化；单 Agent 10 秒超时；SSE 流式展示进度 |
| Docker / Neo4j 部署受阻 | 中 | 高 | 当前已用 MemoryGraph fallback；答辩前必须在干净环境完成 Docker 验证 |
| 知识图谱数据不完整 | 中 | 中 | 优先覆盖 30 个核心节点；其他节点答辩时说明为扩展预留 |
| Pyodide 加载慢 | 中 | 中 | 使用 CDN；首次加载时显示进度条；准备后端 Docker 沙箱 fallback |
| GitHub 推送网络问题 | 低 | 中 | 已配置远程 `origin`；网络恢复后手动/自动执行 `git push -u origin main` |
| 前端组件开发超时 | 中 | 低 | ✅ 已用 Bento + Glassmorphism 完成核心界面重构 |
| 演示视频超时 | 低 | 低 | 精确脚本到秒；准备 1.2 倍速剪辑版 |
