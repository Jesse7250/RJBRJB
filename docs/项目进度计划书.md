# 智学蜂巢 EduHive —— 项目进度计划书

> 赛事：第十五届中国软件杯大赛 A组  
> 赛题：基于大模型的个性化资源生成与学习多智能体系统开发  
> 团队：智学蜂巢项目组  
> 周期：4 周  
> 版本：v1.0

---

## 一、项目目标

在 4 周内完成一个可运行、可演示、具备国一竞争力的 **Python 个性化学习多智能体系统**。系统以 **讯飞星火大模型** 为主推理引擎，以 **Neo4j 知识图谱** 为神经符号约束基础，实现对话式画像构建、多 Agent 协同资源生成、辩论议会质量审核、认知风格自适应渲染等核心功能。

---

## 二、技术路线与边界

### 2.1 核心架构

- **前端**：React 18 + Vite + TypeScript + Tailwind CSS + shadcn/ui
- **后端**：FastAPI + Python 3.10+
- **大模型**：讯飞星火 4.5 Max / Spark Pro
- **多智能体**：自定义 Agent 编排（简化版 AutoGen 思路）
- **知识图谱**：Neo4j Community，40+ 核心知识点
- **数据库**：SQLite（用户、会话、学习记录）
- **代码执行**：Pyodide 浏览器沙箱
- **部署**：Docker Compose

### 2.2 功能边界

| 模块 | 必须完成 | 简化处理 |
|------|---------|---------|
| 对话式画像 | ✅ 6 维度画像 | 初期用规则 + LLM 推断 |
| 多智能体协同 | ✅ 6 个核心 Agent | 辩论议会 4-Agent 必须真实 |
| 5 种资源类型 | ✅ 文档/导图/练习/代码案例/音频 | 视频用 Mermaid 动画 + TTS 替代 |
| 学习路径规划 | ✅ A* + BKT | BKT 先做简化版 |
| 智能辅导 | ✅ 苏格拉底提问链 | 4 轮提问 + 一键答案 |
| 学习效果评估 | ✅ 掌握度热力图 | 基于练习正确率 + 行为数据 |
| 防幻觉 | ✅ 知识图谱约束 + AST 校验 + Guardian | 三层机制完整 |
| 多模态视频 | ⚠️ 用动态图解 + TTS 呈现 | 不追求真实视频生成 |

## 六、风险与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
| 讯飞 API 不稳定 | 高 | 中 | 申请足额额度；本地缓存热门资源；准备离线降级模式 |
| Agent 响应时间过长 | 高 | 中 | 辩论议会异步化；6 轮限制；单 Agent 10 秒超时 |
| 知识图谱数据不完整 | 中 | 中 | 优先覆盖 30 个核心节点；其他节点答辩时说明为扩展预留 |
| Pyodide 加载慢 | 中 | 中 | 使用 CDN；首次加载时显示进度条；准备后端 Docker 沙箱 fallback |
| 前端组件开发超时 | 中 | 中 | 优先保证核心界面；复杂动效可延后 |
| 演示视频超时 | 低 | 低 | 精确脚本到秒；准备 1.2 倍速剪辑版 |
