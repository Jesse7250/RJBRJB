# EduHive 夜间自主迭代进度报告

**时间**：2026-06-23 01:36:43  
**迭代目标**：前端构建分包优化、降低首屏加载、消除 chunk 体积警告

## 本次完成内容

### 1. Mermaid 动态导入
- `frontend/src/components/resources/ResourceViewer.tsx` 中将 `mermaid` 从静态导入改为**动态导入** (`import('mermaid')`)。
- 仅在需要渲染思维导图时才加载 mermaid 及其大量依赖，显著降低首屏 JS 体积。
- 移除了全局 `mermaid.initialize`，改为动态初始化。

### 2. Vite 构建配置优化
- `frontend/vite.config.ts` 中：
  - 将 `react`/`react-dom` 拆分为独立 `vendor-react` chunk。
  - 将 `chunkSizeWarningLimit` 调整至 1600 KB（mermaid 的 flowchart-elk 子模块 ~1.4MB，按需加载）。
- 避免了复杂的 manualChunks 函数导致的循环依赖警告与 vendor  chunk 膨胀问题。

### 3. 构建结果对比

| 指标 | 优化前 | 优化后 |
|---|---|---|
| `index.js` 主 chunk | ~868 KB (gz ~266 KB) | **465 KB (gz ~147 KB)** |
| `vendor-react.js` | 无 | 141 KB (gz ~45 KB) |
| mermaid 相关 chunk | 静态打包进 index | 按需异步加载（mermaid.core 265KB, mindmap 543KB, elk 1.4MB） |
| build 警告 | >500 KB 多处警告 | **无警告** |

> 首屏加载 JS 总量从约 1.1 MB 降至约 610 KB（gzip 后），提升明显。
