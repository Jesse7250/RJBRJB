# EduHive 夜间自主迭代进度报告

**时间**：2026-06-23 02:20:23  
**迭代目标**：JWT 用户认证体系

## 本次完成内容

### 1. 认证服务
- 新增 `backend/app/services/auth.py`：
  - 使用 `bcrypt` 直接进行密码哈希与校验（绕开 passlib 与新版 bcrypt 的兼容性 bug）。
  - JWT token 生成与解析（HS256，24 小时有效期）。
  - 提供 `get_current_user`（可选登录）与 `require_user`（强制登录）依赖。

### 2. 用户数据表
- `backend/app/services/database.py`：
  - 新增 `users` 表（username PK、hashed_password、created_at）。
  - 新增 `create_user`、`get_user` 函数。

### 3. 认证 API
- 新增 `backend/app/api/auth.py`：
  - `POST /api/auth/register`：用户注册，密码至少 6 位，用户名唯一。
  - `POST /api/auth/login`：OAuth2 Password Bearer 登录，返回 JWT。
- 在 `backend/app/main.py` 注册 `/api/auth` 路由。

### 4. 测试
- 新增 `backend/test_auth.py`：注册 + 登录 + 错误密码测试。
