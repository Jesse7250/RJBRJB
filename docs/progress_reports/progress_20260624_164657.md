## 一、本次合并内容

### 1. 新增 BKT（贝叶斯知识追踪）与学习评估接口

| 文件 | 说明 |
|------|------|
| `backend/app/services/bkt.py` | 标准 BKT 模型实现，支持每个知识点的掌握概率追踪、更新与持久化 |
| `backend/app/api/evaluation.py` | 新增 3 个接口：`/api/evaluation/heatmap`、`/api/evaluation/bkt`、`/api/evaluation/analyze` |

功能：
- 基于练习正确/错误更新知识点掌握概率。
- 返回掌握度热力图数据、薄弱知识点、推荐复习知识点。
- 分析接口会从 `learning_events` 和 `code_submissions` 恢复历史并重新计算掌握度。

### 2. 新增前端行为埋点接口

| 文件 | 说明 |
|------|------|
| `backend/app/models/schemas.py` | 新增 `BehaviorEventType` 枚举与 `BehaviorEventRequest` 模型 |
| `backend/app/api/sessions.py` | 新增 `POST /api/sessions/{session_id}/behavior` |
| `backend/app/services/database.py` | `log_event` 增加 `concept` 参数；新增 `log_behavior_event` 自动转认知证据 |

支持事件：点击导图、运行代码、停留时长、展开提示、切换资源、查看画像/路径/辩论报告、播放音频、尝试答题等。

### 3. 资源版本与全局错误统计（知识熔炉基础）

| 文件 | 说明 |
|------|------|
| `backend/app/services/database.py` | 新增 `save_resource_version`、`get_resource_versions_by_id`、`get_global_error_stats` |

为后续「知识熔炉」群体资源进化提供数据支撑：
- 记录资源版本演进（v1.0 → v1.1 → v1.2）。
- 统计某知识点的全局代码提交错误率。

### 4. 合并前修复的问题

在合并到 `main` 之前，修复了同学 PR 中的 3 个阻塞问题：

| 问题 | 修复 |
|------|------|
| `get_resource_versions` 同名函数覆盖 | 改名为 `get_resource_versions_by_id(resource_id)`，保留原有按 `concept` 查询的函数 |
| `health_detail` 统计表名用复数 | 改为实际表名：`resource`、`generation_task`、`debate_record`、`code_submission`、`resource_version` |
| BKT 全局单例导致会话串扰 | `get_bkt_tracker()` 每次返回新实例，`load_from_session()` 前清空模型 |
| BKT 分析结果未持久化 | `/api/evaluation/analyze` 分析后调用 `persist_to_session(session_id)` |
