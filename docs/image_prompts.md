# 智慧伴学 EduMate 作品设计文档配图提示词（v3.0）

> 本文档对应 `docs/作品设计文档.md` v2.1，最终保留 **14 张图**。  
> 目标：全部使用真实 UI 截图或干净的矢量技术图，避免“AI 生成感”和“文字可替代”的框图。  
> 所有统计图使用 matplotlib / Python 生成；所有架构/流程图使用 PlantUML / draw.io / matplotlib 生成；界面图直接使用端到端测试截图或补截。

---

## 1 配图筛选原则

### 1.1 保留标准（满足至少一条，且尽量多条）

| 标准 | 说明 | 本项目示例 |
|------|------|-----------|
| **真实 UI 截图** | 展示真实组件、按钮、数据、会话 ID | 课程门户、知识图谱、学习资源、代码沙箱、掌握进度 |
| **真实数据** | 出现具体数字、比例、阈值、统计结果 | 33 知识点、掌握度 88%、主包 465 KB → 98 KB |
| **动态变化/前后对比** | 时间轴、状态变化、版本演进 | 资源生成过程、BKT 热力图 |
| **多尺度/多面板** | 主图 + 局部 inset 展示关键细节 | 总体数据闭环四宫格 |
| **量化对比** | 用图表对比优化前后 | 性能柱状图 |

### 1.2 删除标准

- 纯概念场景插画（如“学生围在系统周围”）。
- 用文字/表格即可完全替代的层次图、环形图、雷达图。
- 与正文或其他图重复的流程框图。
- 无真实数据支撑的“竞争优势雷达图”。

### 1.3 统一视觉规范

**色彩系统**

| 角色 | 色值 | 用途 |
|------|------|------|
| 主蓝 | `#0F4D92` | 核心系统、Orchestrator |
| 辅蓝 | `#3775BA` | API 层、辅助信息 |
| 青绿 | `#42949E` | 知识图谱、学习路径、数据层 |
| 草绿 | `#8BCF8B` | 通过/安全/正向 |
| 紫红 | `#9A4D8E` | 资源生成、服务层 |
| 砖红 | `#B64342` | 审核/校验/错误/拒绝 |
| 金色 | `#FFD700` | 高亮路径、重点节点 |
| 深灰 | `#4D4D4D` | 主要文字、边框 |
| 浅灰 | `#CFCECE` / `#F7F8FA` | 背景、网格、非关键边 |

**字体与字号**

- 中文：`Source Han Sans SC` / `Microsoft YaHei`；英文：`Arial` / `Helvetica`。
- 图内标题 14–16 pt，模块名称 11–12 pt，正文标注 9–10 pt，辅助标签 8 pt。
- 文字保留为可编辑对象，禁止栅格化。

**线条与布局**

- 主流程线 2 pt，辅助线 1 pt，高亮路径 3 pt 金色。
- 对齐到 4/8 pt 网格；图四周留 24–32 pt 边距。
- 单图主色 ≤ 3 种，辅助色 ≤ 2 种；白底或极浅灰底，禁止纹理/渐变背景。

**输出格式**

- 矢量图：SVG（主文件）+ PDF（印刷备用）+ PNG（预览）。
- 截图：PNG，宽度 ≥ 1920 px，必要时局部裁剪并加 1 pt 细边框。

### 图 6：神经符号约束流程

- **文件名**：`docs/images/fig6-neuro-symbolic.svg`（+ .pdf/.png）
- **类型**：矢量流程图
- **内容**：三阶段水平流程：
  1. **生成前约束**：从知识图谱提取“前置知识、难度、易错点、禁止概念、允许模块”。
  2. **生成中**：Generator 调用 LLM 生成资源（文档/练习/代码）。
  3. **生成后校验**：AST 语法检查 → 导入白名单 → 未学概念检测。
- **绘制要求**：
  - 每个阶段用矩形框，框内列出真实条目（例如目标知识点“文件操作”，禁止概念“类与对象”）。
  - 在 Generator 框内示例一段被约束后的 Python 代码片段。
  - 在右侧校验框标注“通过/未通过”状态。
  - 使用主蓝、青绿、砖红三色区分阶段。
