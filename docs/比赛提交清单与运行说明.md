# 智慧伴学 EduMate 比赛提交清单与运行说明

本文档对应比赛要求中的“可完整运行的多智能体相关文件：包含但不限于项目源码、数据集、模型部署配置文件等，文件整理规范，可在常规环境下正常运行”。

## 一、建议提交内容

建议将仓库根目录 `RJBRJB` 作为完整源码提交，至少包含以下内容：

1. `backend/`
   - `backend/app/`：FastAPI 后端源码，包括认证、学习会话、知识图谱、资源生成、代码执行、掌握度评估、教师端、管理端、数字人助教、TTS 等接口。
   - `backend/app/agents/`：多智能体实现，包括学习画像、路径规划、资源生成、审核、苏格拉底式引导等逻辑。
   - `backend/requirements.txt`：Python 依赖清单。
   - `backend/.env.example`：环境变量模板，只包含字段名和示例值。
   - `backend/Dockerfile`：后端容器配置。
   - `backend/tests/`：后端测试用例。
   - `backend/uploads/`：教师上传资源目录，可保留示例文件或空目录说明。

2. `frontend/`
   - `frontend/src/`：React + TypeScript 前端源码，包括课程广场、学生端、教师端、管理端与数字人交互。
   - `frontend/public/`：页面静态素材、课程图、数字人形象等。
   - `frontend/package.json`、`frontend/package-lock.json`：Node 依赖与锁定版本。
   - `frontend/vite.config.ts`、`frontend/tsconfig*.json`、`frontend/tailwind.config.js`、`frontend/postcss.config.js`：前端构建配置。
   - `frontend/Dockerfile`：前端容器配置。

3. `data/`
   - 知识图谱种子数据或课程知识点数据，用于说明图谱来源和初始化内容。

4. `docs/`
   - 项目计划书、分工文档、接口说明、测试报告、更新报告、演示稿和本提交说明。

5. 根目录配置
   - `README.md`：项目简介和快速启动。
   - `SETUP.md`：详细环境搭建。
   - `environment.yml`：Conda 环境配置。
   - `docker-compose.yml`：一键容器化运行配置。
   - `.gitignore`：提交排除规则。
   - `scripts/start_backend.ps1`、`scripts/start_frontend.ps1`：Windows 本地启动脚本。

## 二、不建议提交内容

以下内容不应提交到 GitHub 或比赛压缩包：

1. 密钥与个人配置
   - `backend/.env`
   - `.env`
   - 真实的 `DEEPSEEK_API_KEY`
   - 真实的讯飞 `APP_ID`、`API_KEY`、`API_SECRET`
   - 真实的 Neo4j 密码

2. 本机环境与依赖缓存
   - `.conda/`
   - `venv/`
   - `.venv/`
   - `frontend/node_modules/`
   - Python `__pycache__/`
   - `.pytest_cache/`

3. 构建产物与运行数据
   - `frontend/dist/`
   - `backend/*.db`
   - `*.log`
   - 临时 TTS 音频、临时截图、个人备份压缩包

说明：SQLite 数据库是运行时数据，可能包含用户、会话、学习记录、课程编辑记录等，不建议作为源码提交。项目首次运行时会自动初始化所需表结构。

## 三、是否需要提交可执行程序包

一般不需要提交 Windows `.exe`。

本项目是 Web 应用和多智能体系统，推荐提交“源码 + 环境配置 + 启动说明 + 可选 Docker Compose”。这种方式更适合评审在常规环境下复现，也方便检查前后端源码、模型配置和数据文件。

如果比赛平台明确要求“可执行程序包”，建议提交 Docker Compose 方案，而不是打包为单个 `.exe`。原因是项目包含前端服务、后端服务、可选 Neo4j、外部大模型 API 和讯飞 TTS，多服务部署比单文件程序更符合项目形态。

## 四、Windows 本地运行方式

后端端口：`8001`  
前端端口：`5173`

后端：

```powershell
cd I:\project\rjb\RJBRJB
copy backend\.env.example backend\.env
# 编辑 backend\.env，填写 DeepSeek 与讯飞 TTS 配置
powershell -ExecutionPolicy Bypass -File .\scripts\start_backend.ps1
```

前端：

```powershell
cd I:\project\rjb\RJBRJB
powershell -ExecutionPolicy Bypass -File .\scripts\start_frontend.ps1
```

浏览器访问：

```text
http://127.0.0.1:5173/#/portal
```

## 五、手动运行命令

后端：

```powershell
cd I:\project\rjb\RJBRJB\backend
..\.conda\python.exe -m pip install -r requirements.txt
..\.conda\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

前端：

```powershell
cd I:\project\rjb\RJBRJB\frontend
npm install
npm run dev -- --host 127.0.0.1 --port 5173
```

## 六、Docker Compose 运行

```powershell
cd I:\project\rjb\RJBRJB
copy backend\.env.example backend\.env
docker compose up --build
```

运行后访问：

```text
http://127.0.0.1:5173/#/portal
```

## 七、交付前自检命令

后端导入检查：

```powershell
cd I:\project\rjb\RJBRJB\backend
..\.conda\python.exe -m py_compile app\main.py
```

前端构建检查：

```powershell
cd I:\project\rjb\RJBRJB\frontend
npm run build
```

接口连通检查：

```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:8001/health/detail"
Invoke-RestMethod -Uri "http://127.0.0.1:8001/api/tts/status"
```

## 八、压缩包结构建议

建议压缩包命名：

```text
智慧伴学EduMate_多智能体个性化学习系统_源码包.zip
```

建议压缩包内部结构：

```text
RJBRJB/
├─ backend/
├─ frontend/
├─ data/
├─ docs/
├─ scripts/
├─ README.md
├─ SETUP.md
├─ environment.yml
├─ docker-compose.yml
└─ .gitignore
```

不要把 `.git/`、`backend/.env`、`.conda/`、`frontend/node_modules/`、`frontend/dist/`、数据库和日志文件压入提交包。
